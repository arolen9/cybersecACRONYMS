Cybersecurity Acronyms

Security Fundamentals & Concepts
2FA
Two-factor Authentication: A security process in which a user provides two different authentication factors to verify themselves.
AAA
Authentication, Authorization, and Accounting: A framework for controlling access, enforcing policies, and auditing usage for computer network resources.
ACL
Access Control List: A list of permissions associated with a system resource (object); a set of rules that defines permissions for users or systems to access specific resources, such as files, directories, or network devices.
CAPTCHA
Completely Automated Public Turing Test to Tell Computers and Humans Apart: A challenge-response test used to determine whether the user is human.
CIA
Confidentiality, Integrity, Availability: The three main goals of information security.
DAC
Discretionary Access Control: An access control model where the owner of a resource can control who has access to it.
DEP
Data Execution Prevention: A system-level memory protection feature that marks certain memory areas as non-executable.
DLP
Data Loss Prevention: A set of tools and processes used to ensure that sensitive data is not lost, misused, or accessed by unauthorized users.
FACL
File System Access Control List: An access control list specifically for file systems.
FIM
File Integrity Management: The process of validating the integrity of operating system and application software files.
GPO
Group Policy Object: A collection of settings that define how a system looks and behaves for a defined group of users.
IAM
Identity and Access Management: A framework of policies and technologies for ensuring that the right people have the appropriate access to technology resources.
MAC (Mandatory Access Control)
Mandatory Access Control: An access control model where the operating system or security kernel restricts the ability of a subject to access or perform some operation on an object.
MDM
Mobile Device Management: Software that allows organizations to centrally control, secure, and manage mobile devices.
MFA
Multifactor Authentication: An authentication method that requires the user to provide two or more verification factors to gain access to a resource.
OAUTH
Open Authorization: An open standard for access delegation, commonly used as a way for Internet users to grant websites or applications access to their information on other websites without giving them the passwords.
PAM (Privileged Access Management)
Privileged Access Management: Cybersecurity strategies and technologies for exercising control over the privileged access of users, accounts, processes, and systems across an IT environment.
PHI
Personal Health Information: Any information about health status, provision of health care, or payment for health care that is created or collected by a "covered entity" (or a business associate of a covered entity) and can be linked to a specific individual.
PII
Personally Identifiable Information: Information that can be used to distinguish or trace an individual's identity.
PIV
Personal Identity Verification: A smart card that provides multi-factor authentication.
PCI DSS
Payment Card Industry Data Security Standard: A set of security standards designed to ensure that all companies that process, store, or transmit credit card information maintain a secure environment.
Business Docs and Compliance
AUP
Acceptable Use Policy: A document outlining the acceptable uses of an organization's computer systems and network resources.
BYOD
Bring Your Own Device: A policy that allows employees to use their personal mobile devices to access enterprise data and systems.
CYOD
Choose Your Own Device: A mobile device management strategy where employees choose from a list of corporate-approved devices for work.
COBO
Corporate-owned, Business-only: A mobile device management strategy where devices are owned by the organization and used exclusively for business purposes.
COPE
Corporate Owned, Personally Enabled: A mobile device management strategy where devices are owned by the organization but can be used for personal activities.
EULA
End User License Agreement: A legal contract between a software manufacturer or publisher and the user of the software.
NDA
Non-disclosure Agreement: A legal contract between at least two parties that outlines confidential material, knowledge, or information that the parties wish to share with one another for certain purposes, but wish to restrict access to or by third parties.
GDPR
General Data Protection Regulation: A regulation in EU law on data protection and privacy in the European Union and the European Economic Area.
Cryptography & Encryption
3DES
Triple Data Encryption Standard: A symmetric-key block cipher that applies the DES algorithm three times to each data block; Block ciphers (e.g. AES (Advanced Encryption Standard)) are a type of symmetric encryption that transforms plaintext into ciphertext by processing the plaintext in blocks of fixed size. Stream ciphers (e.g. PRG-based stream ciphers) encrypt data continuously, one character or bit at a time, making them efficient for applications where data is generated in real-time.
AES
Advanced Encryption Standard: A symmetric-key encryption standard adopted by the U.S. government.
AES-256
Advanced Encryption Standards 256-bit: A version of the Advanced Encryption Standard (AES) that uses a 256-bit key length.
ASLR
Address Space Layout Randomization: A security technique that randomly arranges the address space positions of key data areas to protect systems from certain types of cyberattacks, particularly memory corruption vulnerabilities like buffer overflows.
CA
Certificate Authority: A trusted entity that issues digital certificates.
CBC
Cipher Block Chaining: A mode of operation for block ciphers that uses an initialization vector (IV) and links each ciphertext block to the previous one.
CCMP
Counter Mode/CBC-MAC Protocol: An encryption protocol designed for wireless LAN products that implements AES.
CFB
Cipher Feedback: A mode of operation for block ciphers that makes them into stream ciphers by using the previous ciphertext block as input to the encryption algorithm.
CRL
Certificate Revocation List: A list of digital certificates that have been revoked by the issuing Certificate Authority before their scheduled expiration.
CSR
Certificate Signing Request: A message sent from an applicant to a certificate authority to apply for a digital identity certificate.
CTM
Counter Mode: A mode of operation for block ciphers that turns a block cipher into a stream cipher.
DES
Digital Encryption Standard: A symmetric-key algorithm for the encryption of electronic data.
DHE
Diffie-Hellman Ephemeral: A variant of the Diffie-Hellman key exchange that generates a new key pair for each session.
DSA
Digital Signature Algorithm: A Federal Information Processing Standard (FIPS) for digital signatures.
ECB
Electronic Code Book: The simplest mode of operation for block ciphers, where each block is encrypted independently.
ECC
Elliptic Curve Cryptography: A public-key cryptography approach based on the algebraic structure of elliptic curves over finite fields.
ECDHE
Elliptic Curve Diffie-Hellman Ephemeral: An ephemeral key exchange protocol that uses elliptic curve cryptography.
ECDSA
Elliptic Curve Digital Signature Algorithm: A variant of the Digital Signature Algorithm (DSA) which uses elliptic curve cryptography.
EFS
Encrypted File System: A feature of Windows that allows for transparent encryption and decryption of files on NTFS volumes.
FDE
Full Disk Encryption: A technology that encrypts all data on a disk drive.
GCM
Galois Counter Mode: A mode of operation for block ciphers that provides both confidentiality and authentication.
GPG
Gnu Privacy Guard: A free and open-source implementation of the OpenPGP standard.
HMAC
Hashed Message Authentication Code: A specific type of message authentication code (MAC) involving a cryptographic hash function and a secret cryptographic key.
HOTP
HMAC-based One-time Password: An algorithm for generating one-time passwords based on HMAC.
HSM
Hardware Security Module: A physical computing device that safeguards and manages digital keys for strong authentication and provides cryptoprocessing.
IDEA
International Data Encryption Algorithm: A symmetric-key block cipher that uses a 128-bit key.
IV
Initialization Vector: A fixed-size input to a cryptographic primitive that is typically required to be random or pseudo-random.
KDC
Key Distribution Center: A network service that issues session tickets and temporary session keys to users and computers in an Active Directory domain.
KEK
Key Encryption Key: A key used to encrypt other keys.
MD5
Message Digest 5: A widely used cryptographic hash function producing a 128-bit (16-byte) hash value.
OCSP
Online Certificate Status Protocol: An Internet protocol used for obtaining the revocation status of an X.509 digital certificate.
P12
PKCS #12: A standard for storing private keys with their corresponding public key certificates.
PBKDF2
Password-based Key Derivation Function 2: A key derivation function that reduces the vulnerability of encrypted keys to brute-force attacks.
PEM
Privacy Enhanced Mail: A standard for encrypting email.
PFS
Perfect Forward Secrecy: A feature of a key agreement protocol that gives assurances that session keys will not be compromised even if the long-term secret key used to derive them is compromised.
PGP
Pretty Good Privacy: A data encryption and decryption computer program that provides cryptographic privacy and authentication for data communication.
PKCS
Public Key Cryptography Standards: A group of standards designed by RSA Laboratories to ensure compatibility in public-key cryptography.
PKI
Public Key Infrastructure: A set of roles, policies, and procedures needed to create, manage, distribute, use, store, and revoke digital certificates and manage public-key encryption.
Network & Communication Protocols
AH
Authentication Header: A protocol that provides connectionless integrity and data origin authentication for IP datagrams and provides protection against replays; a security protocol used within the IPsec suite to ensure data integrity, data origin authentication, and optional anti-replay protection for IP packets. It verifies that the data was not modified during transmission and confirms the identity of the sender.
AP
Access Point: A device that allows wireless devices to connect to a wired network.
API
Application Programming Interface: A set of defined rules that enable different applications to communicate with each other.
ARP
Address Resolution Protocol: A protocol used to resolve an IP address to a physical MAC address.
BGP
Border Gateway Protocol: A standardized exterior gateway protocol designed to exchange routing and reachability information among autonomous systems (AS) on the Internet.
BPDU
Bridge Protocol Data Unit: Data messages exchanged between switches to detect loops and build a spanning tree in a network.
CHAP
Challenge Handshake Authentication Protocol: An authentication protocol that periodically verifies the identity of the client by using a three-way handshake.
CSU
Channel Service Unit: A digital interface device that connects a data terminal equipment (DTE) device to a digital circuit.
DHCP
Dynamic Host Configuration Protocol: A network management protocol used on Internet Protocol (IP) networks for dynamically distributing network configuration parameters.
DKIM
DomainKeys Identified Mail: An email authentication method designed to detect forged sender addresses in emails.
DNAT
Destination Network Address Translation: A type of NAT where the destination IP address of a packet is changed.
DNS
Domain Name System: A hierarchical and decentralized naming system for computers, services, or other resources connected to the Internet or a private network.
DNSSEC
Domain Name System Security Extensions: A suite of Internet Engineering Task Force (IETF) specifications for securing certain kinds of information provided by the Domain Name System (DNS).
DMARC
Domain Message Authentication Reporting and Conformance: An email authentication protocol designed to protect against email spoofing.
DSL
Digital Subscriber Line: A family of technologies that are used to transmit digital data over telephone lines.
EAP
Extensible Authentication Protocol: An authentication framework that supports multiple authentication methods.
ESP
Encapsulated Security Payload: A protocol that provides confidentiality, data origin authentication, connectionless integrity, an anti-replay service, and limited traffic flow confidentiality for IP datagrams.
FTP
File Transfer Protocol: A standard network protocol used for the transfer of computer files from a server to a client on a computer network.
FTPS
Secured File Transfer Protocol: An extension of FTP that adds support for the Transport Layer Security (TLS) and Secure Sockets Layer (SSL) cryptographic protocols.
GRE
Generic Routing Encapsulation: A tunneling protocol that can encapsulate a wide variety of network layer protocols inside virtual point-to-point links over an Internet Protocol internetwork.
HTTP
Hypertext Transfer Protocol: The foundation of data communication for the World Wide Web.
HTTPS
Hypertext Transfer Protocol Secure: An extension of the Hypertext Transfer Protocol (HTTP) for secure communication over a computer network.
ICMP
Internet Control Message Protocol: A supporting protocol in the Internet Protocol suite.
IKE
Internet Key Exchange: The protocol used to set up a security association in the IPsec protocol suite.
IM
Instant Messaging: A type of online chat that offers real-time text transmission over the Internet.
IMAP
Internet Message Access Protocol: A standard protocol for accessing email messages from a mail server.
IP
Internet Protocol: The principal communications protocol in the Internet protocol suite for relaying datagrams across network boundaries.
IPSec
Internet Protocol Security: A suite of protocols for securing Internet Protocol (IP) communications by authenticating and encrypting each IP packet in a data stream.
IRC
Internet Relay Chat: A text-based chat system that allows real-time communication among large groups of users.
L2TP
Layer 2 Tunneling Protocol: A tunneling protocol used to support virtual private networks (VPNs) or as part of the delivery of services by ISPs.
LAN
Local Area Network: A computer network that interconnects computers within a limited area such as a residence, school, laboratory, university campus, or office building.
LDAP
Lightweight Directory Access Protocol: An open, vendor-neutral, industry standard application protocol for accessing and maintaining distributed directory information services.
LEAP
Lightweight Extensible Authentication Protocol: A Cisco proprietary version of EAP used for wireless authentication.
MAC (Media Access Control)
Media Access Control: A unique identifier assigned to network interfaces for communications within a network segment.
MAC (Message Authentication Code)
Message Authentication Code: A short piece of information used to authenticate a message and to provide integrity and authenticity.
MAN
Metropolitan Area Network: A computer network that interconnects users with computer resources in a geographic area or region larger than that covered by even a large local area network (LAN) but smaller than the area covered by a wide area network (WAN).
MMS
Multimedia Message Service: A standard way to send messages that include multimedia content to and from mobile phones over a cellular network.
MPLS
Multi-protocol Label Switching: A routing technique in telecommunications networks that directs data from one network node to the next based on short path labels rather than long network addresses.
MSCHAP
Microsoft Challenge Handshake Authentication Protocol: Microsoft's proprietary version of the Challenge Handshake Authentication Protocol (CHAP).
MTU
Maximum Transmission Unit: The largest packet size that can be transmitted over a network link without fragmentation.
NAC
Network Access Control: A computer networking solution that uses a set of protocols to define and implement a policy that describes how to secure network nodes when they initially attempt to access the network.
NAT
Network Address Translation: A method of remapping one IP address space into another by modifying network address information in the IP header of packets while they are in transit across a traffic routing device.
NFC
Near Field Communication: A set of communication protocols for communication between two electronic devices over a distance of 4 cm or less.
NGFW
Next-generation Firewall: A part of the third generation of firewall technology, combining a traditional firewall with other network device filtering functions.
NTLM
New Technology LAN Manager: A suite of Microsoft security protocols that provide authentication, integrity, and confidentiality to users.
NTP
Network Time Protocol: A networking protocol for clock synchronization between computer systems over packet-switched, variable-latency data networks.
OSPF
Open Shortest Path First: A routing protocol for Internet Protocol (IP) networks.
OTA
Over the Air: A method of distributing new software, configuration settings, and even updating encryption keys to devices like cellphones.
PAC
Proxy Auto Configuration: A method that allows web browsers and other user agents to automatically choose the appropriate proxy server for a given URL.
PAP
Password Authentication Protocol: A simple authentication protocol used by Point-to-Point Protocol (PPP) to validate users.
PAT
Port Address Translation: A type of Network Address Translation (NAT) that maps multiple private IP addresses to a single public IP address by using different port numbers.
PBX
Private Branch Exchange: A private telephone network used within a company or organization.
POP
Post Office Protocol: An application-layer Internet standard protocol used by local email clients to retrieve email from a remote server over a TCP/IP connection.
POTS
Plain Old Telephone Service: The traditional analog telephone system.
PPP
Point-to-Point Protocol: A data link layer (Layer 2) communication protocol used to establish a direct connection between two nodes.
PPTP
Point-to-Point Tunneling Protocol: An obsolete method for implementing virtual private networks.
Cyber Threats & Vulnerabilities
APT
Advanced Persistent Threat: A sophisticated, prolonged cyberattack where an intruder gains access to a network and stays there undetected for a long period of time.
ATT&CK
Adversarial Tactics, Techniques, and Common Knowledge: A globally accessible knowledge base of adversary tactics and techniques based on real-world observations.
CSRF
Cross-site Request Forgery: An attack that tricks a web browser into executing an unwanted action on a trusted site where the user is authenticated.
CVE
Common Vulnerability Enumeration: A list of publicly disclosed cybersecurity vulnerabilities.
CVSS
Common Vulnerability Scoring System: A free and open industry standard for assessing the severity of computer system security vulnerabilities.
DDoS
Distributed Denial of Service: A cyberattack where multiple compromised computer systems attack a target, such as a server, website, or other network resource, and cause a denial of service for users of the targeted resource.
DoS
Denial of Service: A cyberattack where the perpetrator seeks to make a machine or network resource unavailable to its intended users.
IoC
Indicators of Compromise: Forensic data found on a network or operating system that indicates a probable intrusion.
OSINT
Open-source Intelligence: Intelligence gathered from publicly available sources.
OVAL
Open Vulnerability Assessment Language: An international, public, and community effort to standardize how to assess and report upon the machine state of computer systems.
Security Operations & Management
AIS
Automated Indicator Sharing: A system for automated sharing of cyber threat indicators and defensive measures; Automated Indicator Sharing (AIS) is a service provided by the Cybersecurity and Infrastructure Security Agency (CISA) that facilitates the exchange of machine-readable cyber threat indicators (CTIs) and defensive measures (DMs) between public and private sector organizations. This initiative aims to improve the cybersecurity posture of participants by sharing information about potential threats and vulnerabilities.
ALE
Annualized Loss Expectancy: The expected monetary loss for an asset due to a risk over a one-year period.
ARO
Annualized Rate of Occurrence: The probability that a specific threat will occur in a single year.
BCP
Business Continuity Planning: The process of creating systems of prevention and recovery to deal with potential threats to a company.
BIA
Business Impact Analysis: A systematic process to determine and evaluate the potential effects of an interruption to critical business operations.
BPA
Business Partners Agreement: A legal agreement between two or more partners that outlines the terms and conditions of their partnership.
CAR
Corrective Action Report: A formal document used to record details of issues, their root causes, and the actions taken to correct them.
CASB
Cloud Access Security Broker: Software or hardware that acts as an intermediary between users and cloud service providers.
CERT
Computer Emergency Response Team: An expert group that handles computer security incidents.
CIO
Chief Information Officer: A senior executive who is responsible for managing and overseeing the information technology (IT) systems of an organization.
CIRT
Computer Incident Response Team: A team responsible for responding to computer security incidents.
COOP
Continuity of Operation Planning: Efforts to ensure that essential functions continue during a wide range of emergencies or disasters.
CP
Contingency Planning: The process of preparing for unforeseen events or disasters.
CSO
Chief Security Officer: A senior-level executive responsible for an organization's security.
CSP
Cloud Service Provider: A third-party company that offers cloud-based computing services to customers.
CTO
Chief Technology Officer: A senior executive who is responsible for the overall technology strategy of an organization.
DBA
Database Administrator: A person responsible for the design, implementation, maintenance, and repair of an organization's databases.
DPO
Data Privacy Officer: A person responsible for overseeing data protection strategy and implementation to ensure compliance with GDPR requirements.
DRP
Disaster Recovery Plan: A documented process or set of procedures to recover and protect a business IT infrastructure in the event of a disaster.
HA
High Availability: A system or component that is continuously operational for a desirably long length of time.
HIDS
Host-based Intrusion Detection System: An intrusion detection system that monitors and analyzes the internals of a computing system as well as the network packets on its network interfaces.
HIPS
Host-based Intrusion Prevention System: An intrusion prevention system that monitors a single host for suspicious activity.
IdP
Identity Provider: A system entity that creates, maintains, and manages identity information for principals and provides authentication services to relying party applications.
IDS
Intrusion Detection System: A device or software application that monitors a network or systems for malicious activity or policy violations.
IR
Incident Response: An organized approach to addressing and managing the aftermath of a security breach or cyberattack.
IRP
Incident Response Plan: A documented plan for responding to security incidents.
ISO
International Standards Organization: An international standard-setting body composed of representatives from various national standards organizations.
ISP
Internet Service Provider: An organization that provides services for accessing, using, or participating in the Internet.
ISSO
Information Systems Security Officer: A person responsible for the overall information security posture of a system or organization.
MOA
Memorandum of Agreement: A document that describes the terms and conditions of a mutual understanding between two or more parties.
MOU
Memorandum of Understanding: A formal agreement between two or more parties.
MSA
Master Service Agreement: A contract that specifies the terms that will govern future agreements between parties.
MSP
Managed Service Provider: A company that remotely manages a customer's IT infrastructure and/or end-user systems.
MSSP
Managed Security Service Provider: An organization that provides outsourced monitoring and management of security devices and systems.
MTBF
Mean Time Between Failures: The predicted elapsed time between inherent failures of a mechanical or electronic system during normal system operation.
MTTF
Mean Time to Failure: The average time a device or system is expected to function before it fails completely.
MTTR
Mean Time to Recover: The average time it takes to repair a failed system or component.
NIDS
Network-based Intrusion Detection System: An intrusion detection system that monitors network traffic to detect suspicious activity.
NIPS
Network-based Intrusion Prevention System: An intrusion prevention system that monitors network traffic to detect and prevent suspicious activity.
NIST
National Institute of Standards & Technology: A physical sciences laboratory and non-regulatory agency of the United States Department of Commerce.
Hardware, Software & Infrastructure
AI
Artificial Intelligence: The simulation of human intelligence processes by machines.
AV
Antivirus: Software designed to detect, prevent, and remove malicious software.
BASH
Bourne Again Shell: A Unix shell and command language interpreter.
BIOS
Basic Input/Output System: Firmware used to perform hardware initialization during the booting process and to provide runtime services for operating systems and programs.
CCTV
Closed-circuit Television: A video system that is locally monitored and not broadcast publicly.
CMS
Content Management System: A software application or set of related programs used to create and manage digital content.
CRC
Cyclical Redundancy Check: An error-detecting code used to detect accidental changes to raw data.
DLL
Dynamic Link Library: A Microsoft implementation of the shared library concept.
EDR
Endpoint Detection and Response: A cybersecurity solution that continuously monitors and responds to cyber threats on endpoints.
ERP
Enterprise Resource Planning: A system that integrates all facets of an operation, including product planning, development, manufacturing, sales, and marketing.
ESN
Electronic Serial Number: A unique identification number embedded in a mobile phone by the manufacturer.
FPGA
Field Programmable Gate Array: An integrated circuit designed to be configured by a customer or a designer after manufacturing.
GPS
Global Positioning System: A satellite-based navigation system that provides location and time information.
GPU
Graphics Processing Unit: A specialized electronic circuit designed to rapidly manipulate and alter memory to accelerate the creation of images in a frame buffer intended for output to a display device.
HDD
Hard Disk Drive: An electromechanical data storage device that stores and retrieves digital data using rapidly rotating platters coated with magnetic material.
HTML
Hypertext Markup Language: The standard markup language for documents designed to be displayed in a web browser.
HVAC
Heating, Ventilation Air Conditioning: Technologies for indoor and vehicular environmental comfort.
IaaS
Infrastructure as a Service: A cloud computing service model that provides virtualized computing resources over the internet.
IaC
Infrastructure as Code: The managing and provisioning of computer data centers through machine-readable definition files.
ICS
Industrial Control Systems: Computerized systems that control industrial processes.
IDF
Intermediate Distribution Frame: A cable rack used to connect telephone company lines to the internal network.
IoT
Internet of Things: The inter-networking of physical devices, vehicles, home appliances, and other items embedded with electronics, software, sensors, actuators, and network connectivity which enable these objects to collect and exchange data.
MaaS
Monitoring as a Service: A cloud-based service that provides monitoring capabilities for various IT infrastructure components.
MBR
Master Boot Record: The first sector of a data storage device.
MDF
Main Distribution Frame: A signal distribution frame or cable rack used to interconnect and manage telecommunication wiring between itself and any number of intermediate distribution frames (IDFs) and equipment.
MFD
Multifunction Device: A device that performs multiple functions, such as printing, scanning, copying, and faxing.
MFP
Multifunction Printer: A device that combines the functionality of a printer, copier, scanner, and/or fax machine.
ML
Machine Learning: A type of artificial intelligence that provides computers with the ability to learn without being explicitly programmed.
NTFS
New Technology File System: The default file system for the Windows NT family of operating systems.
OID
Object Identifier: An identifier mechanism standardized by the ITU-T and ISO for naming any object, concept, or "thing" with an unambiguous name, for use in a global information system.
OS
Operating System: System software that manages computer hardware and software resources and provides common services for computer programs.
OT
Operational Technology: Hardware and software that detects or causes a change through the direct monitoring and/or control of physical devices, processes, and events in the enterprise.
P2P
Peer to Peer: A distributed application architecture that partitions tasks or workloads between peers.
PaaS
Platform as a Service: A cloud computing service model that provides customers with a complete platform to develop, run, and manage applications without the complexity of building and maintaining the infrastructure typically associated with developing and launching an app.
PAM (Pluggable Authentication Modules)
Pluggable Authentication Modules: A mechanism that integrates multiple low-level authentication schemes into a high-level API.
PCAP
Packet Capture: The process of intercepting and logging traffic that passes over a digital network.
PDU
Power Distribution Unit: A device fitted with multiple outputs designed to distribute electric power to computers or networking equipment within a rack.
PED
Personal Electronic Device: Any portable electronic device capable of communication, data processing, or entertainment.
